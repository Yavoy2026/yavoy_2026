# Документация проекта YaVoy Travel Group

**Версия:** 1.0  
**Дата:** 24 июня 2026  
**Платформы:** Expo (React Native), iOS (SwiftUI), Web (React), Android (Kotlin/Compose)  
**Бэкенд:** `https://yavay.ru/backend/api/v1`

---

## 1. О проекте

**YaVoy Travel Group** — кроссплатформенный агрегатор туристических экскурсий по России. Пользователи просматривают, бронируют и добавляют в избранное экскурсии; партнёры (организаторы туров) публикуют свои экскурсии; администраторы модерируют контент и управляют пользователями.

Приложение реализовано на четырёх платформах с единым дизайном и функциональностью.

## 4. Пользовательские функции

### 4.1 Главный экран (Экскурсии)

**Назначение:** поиск и фильтрация экскурсий.

- **Геолокация** (Expo) — автоматическое определение города пользователя
- **Выбор города** — горизонтальная прокручиваемая лента чипов с названиями городов
- **Поиск** — текстовый поиск по названию, описанию, имени организатора и городу
- **Фильтры:**
  - Длительность: однодневные / многодневные
  - Транспорт: авто, водные, морские, вело, авиа
  - Интересы: городские, познавательные, природные, паломничество (+10 категорий в Expo)
  - Категории: агро, фото, этно, родители, глэмпинг, животные, мистика, дикие животные, вино, гастро (Expo)
  - Сезон: зима, весна, лето, осень, круглый год
- **Сортировка:** популярные, новинки, дешевле, дороже
- **Фильтр по дате** (Expo) — выбор конкретной даты поездки
- **Карточки туров** — фото, название, цена, скидка, рейтинг, организатор
- **Карусель популярных туров** — топ-5 по популярности
- **Секция Reels** — блок с видео-историями
- **Баннер сертификатов** (Expo) — покупка подарочных сертификатов
- **Блок преимуществ** — почему выбирают YaVoy (в футере)
- **Pull-to-refresh** (Expo)
- **Сброс фильтров** — кнопка очистки всех фильтров

### 4.2 Экран «Направления»

**Назначение:** обзор городов и категорий.

- **Сетка городов** (2 колонки) — фото, эмодзи, количество туров в городе
- **Фильтр по транспорту** — категории транспорта
- **Топ-туры** — 5-6 лучших экскурсий по рейтингу
- **Результаты фильтрации** — показываются при активном фильтре

### 4.3 Детали тура

**Назначение:** полная информация об экскурсии и бронирование.

- **Галерея изображений** (горизонтальный пейджер с индикаторами)
- **Перелистывание между турами** (Expo: FlatList, iOS: TabView)
- **Цена со скидкой** — оригинальная цена перечёркнута
- **Бейджи:** бестселлер, мгновенное подтверждение, бесплатная отмена, «быстро разбирают»
- **Мета-информация:** город, транспорт, длительность, тип интереса
- **Описание** — полный текст
- **Что взять с собой** — список рекомендаций
- **Что включено / не включено** — две колонки
- **Инфо-карточки:** начало, расписание, размер группы, языки, место встречи
- **Точка сбора** — модальное окно с картой (ссылки на Яндекс.Карты / 2ГИС)
- **Политики:** условия бронирования, предоплата, отмена
- **Карточка организатора** — аватар, рейтинг, верификация, количество туров
- **Рейтинг** — звёзды + количество отзывов
- **Отзывы** — аватар, автор, дата, рейтинг, текст
- **Избранное** — переключатель сердечка
- **Поделиться** — нативное шаринг-меню
- **Бар бронирования** (прилипает снизу) — цена + кнопка «Забронировать»
- **Модальное окно бронирования** — телефон/email, имя, фамилия, количество билетов
- **Бонусные баллы** (Expo) — начисление за покупку
- **Тактильный отклик** (Expo) — вибрация при взаимодействиях

### 4.4 Экран «Избранное»

**Назначение:** просмотр сохранённых туров и городов.

- **Две вкладки:** Туры / Города
- **Список избранных туров** — карточки с возможностью перехода к деталям
- **Карточки городов** — с кнопкой удаления из избранного
- **Навигация к турам города** — при тапе на город открывается главная с фильтром
- **Пустые состояния** — подсказки при отсутствии избранного

### 4.5 Экран «Профиль»

**Назначение:** личный кабинет пользователя.

- **Шапка профиля** — аватар, имя, email, статистика (поездки, баллы, избранное)
- **Гостевой режим** — если не авторизован, показывается кнопка входа
- **Секция «Мои бронирования»** — список с деталями (дата, статус, ваучер, код подтверждения)
- **Секция «Избранное»** — быстрый доступ к сохранённым турам
- **Секция «Мои отзывы»** — список оставленных отзывов
- **Секция «Подарочные сертификаты»** (Expo) — список купленных сертификатов + покупка нового
- **Секция «Промо-коды»** (Expo, Web) — создание кода «пригласи друга» (+2000 баллов)
- **Секция «Мои Reels»** — список отправленных видео со статусом модерации + форма добавления
- **Секция «Транзакции»** — история платежей (сумма, дата, статус)
- **Настройки уведомлений** (Expo) — тоглы: все, новые туры, снижение цен, бронирования, промо-акции
- **Переключатель темы** — Системная / Светлая / Тёмная
- **Кнопка «Для партнёров»** — вход в B2B-кабинет
- **Кнопка «Админ-панель»** — вход для администраторов
- **Модальные окна** — условия использования, о приложении
- **Выход из аккаунта**

### 4.6 Экран Reels

**Назначение:** полноэкранная лента коротких видео из путешествий.

- **Вертикальный пейджер** — перелистывание свайпом (TikTok-стиль)
- **Воспроизведение видео** (Expo: expo-video, Web: HTML5)
- **Кнопка Play/Pause** — по тапу
- **Лайк** — счётчик + анимация
- **Чип города** — название города
- **Информация:** автор, заголовок, название тура, текст истории
- **Счётчик просмотров**
- **Кнопка закрытия** — крестик в углу
- **Индикатор прогресса** — точки или счётчик страниц
- **Учёт просмотров** (Expo) — инкремент при просмотре

### 4.7 Экран «Поддержка» (Expo)

**Назначение:** чат с AI-консультантом по вопросам путешествий.

- **AI-ассистент** на базе Rork Toolkit LLM
- **Системный промпт** — консультант знает все туры, города, политики YaVoy
- **Авто-эскалация** — перевод на живого менеджера при невозможности помочь
- **Чат-интерфейс** — пузыри сообщений (пользователь / AI / агент)
- **Сброс диалога** — кнопка начать заново
- **Клавиатурно-адаптивная панель ввода**

## 5. Партнёрские функции (B2B)

### 5.1 Регистрация партнёра

**Многошаговый процесс:**

1. **Ввод ИНН/ОГРН** — проверка длины (10/12/13/15 цифр), принятие трёх юридических документов (пользовательское соглашение, политика конфиденциальности, договор оферты), мок-верификация ФНС
2. **Контакты** — email, телефон (с префиксом страны), Telegram-никнейм
3. **Ожидание одобрения** — экран статуса
4. **Отклонено** — причина отказа, возможность начать заново
5. **Одобрено** — полный доступ к кабинету

### 5.2 Кабинет партнёра

**5 вкладок:** Туры | Клиенты | Транзакции | Отзывы | Чат

- **Карточка компании** — юр.название, ИНН, ОГРН, тип лица, адрес
- **Рейтинг партнёра** — средняя оценка + количество
- **KPI-карточки:** выручка (за период), гости, опубликовано
- **Фильтр периода:** неделя / месяц / полгода / год / всё время

**Вкладка «Туры»:**
- Список туров с бейджами статуса (на модерации / опубликован / отклонён)
- Форма добавления тура: название, описание, город, цена, медиа, длительность, транспорт, интерес, категория, сезон, размер группы, место встречи
- Выбор фото/видео

**Вкладка «Клиенты»:**
- Список гостей с инициалами, датами, статусом

**Вкладка «Транзакции»:**
- История платежей с суммами, статусами

**Вкладка «Отзывы»:**
- Список отзывов, отображение звёзд
- Форма ответа на отзыв
- Статус ответа (одобрен / на модерации / отклонён)

**Вкладка «Чат» (Expo):**
- Переписка с клиентом + администратором по каждому туру
- Сообщения с ролями: клиент / партнёр / админ

**Просмотр юридических документов** — модальное окно с полным текстом

## 7. Интеграция с бэкендом

### 7.1 API-эндпоинты

**Базовый URL:** `https://yavay.ru/backend/api/v1`

**Аутентификация:**

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/auth/signin` | Вход (email + пароль) → JWT-токены |
| POST | `/auth/signup` | Регистрация (email, пароль, имя) → JWT-токены |
| GET | `/auth/whoami` | Профиль текущего пользователя |
| GET | `/auth/logout` | Выход (инвалидация refresh-токена) |
| GET | `/auth/refresh` | Обновление access-токена |

**Пользователи:**

| Метод | Путь | Описание |
|-------|------|----------|
| PATCH | `/users/{id}` | Обновление профиля (имя, фамилия) |
| PATCH | `/users/{id}/change-password` | Смена пароля |
| PATCH | `/users/{id}/photo` | Загрузка фото профиля |
| PATCH | `/users/{id}/update-role` | Изменение роли (админ) |
| POST | `/users/{id}/activate` | Активация пользователя (админ) |
| DELETE | `/users/{id}/deactivate` | Деактивация пользователя (админ) |

### 7.2 Модель пользователя

```typescript
{
  id: string;
  email: string;
  is_active: boolean;
  role: "admin" | "user" | "moderator";
  first_name: string;
  last_name?: string | null;
  photo?: string | null;
  photo_min?: string | null;
  created_at: string;
  last_login?: string | null;
}
```

### 7.3 Token Management

- **Access token** — короткоживущий, передаётся в заголовке `Authorization: Bearer ...`
- **Refresh token** — долгоживущий, используется для автоматического обновления при 401
- **Хранение:** AsyncStorage (Expo), localStorage (Web), SharedPreferences (Android), UserDefaults (iOS)
- **Авто-рефреш:** при получении 401 система прозрачно обновляет токен и повторяет запрос

- **Сеть:** Ktor Client
